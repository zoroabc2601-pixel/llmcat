# llmcat 🐱

> **Like `cat`, but for AI.** Dump any codebase into perfect LLM context in one command.

```bash
pip install llmcat
llmcat ./my-project -c   # copies entire codebase to clipboard, AI-ready
```

---

Stop manually copy-pasting files into ChatGPT, Claude, or Gemini. `llmcat` does it in one command — formatted perfectly, with a file tree, token estimates, and smart defaults that skip junk (node_modules, .git, binaries, lock files).

## ✨ Features

- **Zero config** — works out of the box on any project
- **Smart filtering** — auto-skips node_modules, .git, binaries, lock files, images
- **Respects `.gitignore`** — automatically reads your gitignore rules
- **Token estimation** — shows ~token count before you paste
- **File tree included** — gives the AI structural context
- **Clipboard support** — `-c` copies directly, no piping needed
- **Save to file** — `-o output.md` for saving context snapshots
- **Flexible input** — pass a directory, specific files, or mix both

## 📦 Install

```bash
pip install llmcat
```

Requires Python 3.10+. No dependencies beyond the standard library.

## 🚀 Usage

```bash
# Current directory → stdout
llmcat .

# Copy to clipboard (macOS/Linux/Windows)
llmcat . -c

# Save to a file
llmcat . -o context.md

# Specific files or subdirectories
llmcat src/ README.md pyproject.toml

# Skip extra folders
llmcat . --ignore tests docs examples

# Limit file count (for huge repos)
llmcat . --max-files 50

# Include hidden files
llmcat . --hidden

# No file tree (just the code)
llmcat . --no-tree
```

## 🧠 Why?

When you ask an AI to help with your code, it needs context. Pasting files one by one is slow. `llmcat` gives it everything — structured, clean, and in one shot.

**Honest note:** tools like [`code2prompt`](https://github.com/mufeedvh/code2prompt) and [`repomix`](https://github.com/yamadashy/repomix) already do this well and have far more history and adoption. `llmcat` doesn't claim to be the first or the best in this space — it's a small, dependency-free, pure-Python alternative I built to learn the problem end to end. If you need something more mature and feature-rich, those are great choices.

**Common workflows:**

```bash
# "Help me refactor this project"
llmcat . -c
# → paste into Claude / ChatGPT

# "Review my PR changes"
llmcat src/ -c

# "Explain this codebase to me"  
llmcat . -o context.md && cat context.md | llm "explain this codebase"

# Save a snapshot before a big refactor
llmcat . -o before-refactor.md
```

## ⚙️ Token Warning

`llmcat` estimates token count and warns you:

```
✓ llmcat — my-project
  Files   : 23
  Chars   : 84,203
  ~Tokens : 21,050
  Copied  : ✓ to clipboard
```

If your project is huge, use `--max-files` or `--ignore` to trim it down.

## 🔧 Supported File Types

`llmcat` includes source code, config files, and docs. It skips:

- `node_modules/`, `__pycache__/`, `.git/`
- Binary files (images, videos, archives, compiled files)
- Lock files (`package-lock.json`, `poetry.lock`, etc.)
- Build artifacts (`dist/`, `build/`, `.next/`)

See [`llmcat/cli.py`](llmcat/cli.py) for the full lists — easy to customize.
